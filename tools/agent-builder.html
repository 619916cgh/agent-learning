🤖 Agent 可视化构建器

通过6步向导配置一个完整的 AI Agent — 选择模型、定义提示词、装配工具、设定安全参数

🧠 选择底层模型

不同模型在推理能力、速度、成本上有显著差异。选择最适合你任务场景的模型。

💬 定义系统提示词

系统提示词定义 Agent 的角色、行为规范和能力边界。

🛠 为 Agent 装配工具

选择 Agent 可以调用的外部工具。工具越多能力越强，但也会增加复杂度和成本。

🧠 记忆系统配置

配置 Agent 的记忆机制，平衡上下文长度和任务连贯性。

较小的窗口节约成本，较大的窗口保留更多上下文

使用向量数据库存储和检索历史对话中的重要信息

🛡 安全与执行参数

设定 Agent 的安全边界，防止过度执行和意外行为。

限制单次任务的最大 Reasoning-Action 循环次数

包括：执行代码、发送请求、修改文件、删除数据、发送邮件等

过滤潜在的提示注入攻击，审核输出内容的安全性

📋 审查与导出配置

仔细检查以下配置，确认无误后导出为 JSON 配置文件。
